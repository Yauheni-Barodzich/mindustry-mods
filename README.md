# mindustry-mods

Моды для **Mindustry v159+**.

Скачать: [Releases](https://github.com/Yauheni-Barodzich/mindustry-mods/releases)

| Архив | Что это |
|-------|---------|
| `dune-start.zip` | Игровой контент: гараж и вездеход-сборщик |
| `sync-admin.zip` | Синхронизация модов/карт с сервера + удалённая админка |

Оба zip ставятся **одинаково** на клиент и на сервер.

---

## Установка

| Куда | Путь |
|------|------|
| Клиент (Windows) | `%AppData%\Mindustry\mods\` |
| Сервер | `config/mods/` рядом с сервером |

Положите оба файла:

```
sync-admin.zip
dune-start.zip
```

Перезапустите игру / сервер.

**Firewall на сервере** (если игроки снаружи): откройте порт игры и два следующих:

| Назначение | Порт (если игра на 6567) |
|------------|---------------------------|
| Игра | `6567` |
| Синхронизация | `6568` (= игра + 1) |
| Админка | `6569` (= игра + 2) |

Карту `DUNE.msav` положите на сервер в `config/maps/` — клиенты скачают её через синхронизацию.

## dune-start — Дюна: Старт

Игровой контент в духе Дюны: ранняя добыча без энергии.

Версия мода на клиенте и на сервере должна совпадать. Если заходите на сервер с этим модом — сначала синхронизируйтесь (см. ниже).

### Гараж вездеходов

- Завод юнитов **без энергии**
- Размер 3×3, стоимость постройки: **50 меди**
- Собирает **вездеход-сборщик** за **5 меди** (~5 секунд)

### Вездеход-сборщик

- Ездит по земле, сам добывает руду с пола
- Трюм **120**, песок сбрасывает
- Разгружается у ядра, контейнера или хранилища (вплотную)
- На команду максимум **3** штуки (лишние удаляются, ресурсы возвращаются)
- Пустой едет быстрее, полный — медленнее

В меню строительства: блок гаража → произвести вездеход.

## sync-admin — Синхронизация и админка

Один пакет на клиент и сервер. В игре появляются две кнопки в **Мультиплеер → Join** и две категории в **Настройки**.

### Синхронизация контента

Сервер раздаёт моды и карты по HTTP. Клиент скачивает недостающее **до** входа на сервер.

#### На сервере

1. `sync-admin.zip` в `config/mods/`
2. Контент-моды (например `dune-start.zip`) тоже в `config/mods/`
3. Карты в `config/maps/`
4. Убедитесь, что открыт порт **игра + 1** (6568)

Конфиг (создаётся сам): `config/mods/server-content-sync/config.hjson`

```
enabled: true
bind: "0.0.0.0"
port: 0          # 0 = порт игры + 1
```

Команды в консоли сервера: `sync-status`, `sync-reload`.

#### В игре (клиент)

1. Поставьте `sync-admin.zip` в `mods/`
2. **Мультиплеер** → **Join** → кнопка **«Синхронизация»**  
   (или **Настройки → Синхронизация → Открыть синхронизацию**)
3. Адрес сервера, например `example.com:6567` (порт **игры**, не 6568)
4. Sync URL можно оставить пустым — подставится `http://хост:6568`
5. **Запросить** — сравнит ваши моды/карты с сервером
6. **Скачать** — скачает недостающие и устаревшие
7. Если обновились **моды** — перезапустите Mindustry, затем заходите на сервер

Локальные моды, которых нет на сервере, **не удаляются** (только показываются в списке).

Через sync **не раздаётся** сам `sync-admin` и серверные плагины — их ставят вручную.

### Админ сервера

Удалённое управление: статус, рестарт, загрузка модов/карт, правила, пресеты, сообщения игрокам.

#### На сервере

1. `sync-admin.zip` в `config/mods/`
2. Задайте пароль **только через FTP/SSH** (не из игры):

   Файл: `config/mods/server-admin/admin.password`  
   Первая непустая строка без `#` — пароль.

3. Откройте порт **игра + 2** (6569)

Конфиг: `config/mods/server-admin/config.hjson`

```
enabled: true
bind: "0.0.0.0"
port: 0                 # 0 = порт игры + 2
sessionTtlSec: 3600
spawnClampEnabled: true
maxEnemySpawns: 3
minEnemySpawns: 1
```

Команды: `admin-status`, `admin-spawn-clamp`.

#### В игре (клиент)

1. **Мультиплеер → Join → «Админ сервера»**  
   (или **Настройки → Админ сервера**)
2. Адрес сервера (`host:6567`), пароль
3. **Войти**

Опционально для быстрого входа: скопируйте тот же пароль в  
`%AppData%\Mindustry\mods\server-admin\admin.password`  
и используйте **«Быстрый вход»**. Пароль в настройках игры не сохраняется.

#### Что можно делать в админке

| Раздел | Действия |
|--------|----------|
| Сервер | Статус, рестарт |
| Сообщение | Чат / Announce / Toast всем онлайн |
| Спавн врагов | Лимит точек спавна после загрузки карты |
| Моды / Карты | Список, загрузка с диска ПК, удаление |
| Правила | Редактор `config/rules.hjson`, применить «на лету» |
| Пресеты | Сохранить/применить наборы правил (нужен рестарт сервера) |
| Конфиг | Сырое чтение/запись файлов конфига |

Шаблоны пресетов: Survival, PvP, Attack. Применение пресета пишет `rules.hjson` и обычно требует **рестарт сервера**.

---

## Типичный сценарий

1. На сервер: `sync-admin.zip`, `dune-start.zip`, карта `DUNE.msav`
2. Пароль в `config/mods/server-admin/admin.password`
3. На клиент: только `sync-admin.zip`
4. Join → **Синхронизация** → запросить → скачать → перезапуск
5. Заход на сервер — появится контент Дюны
6. Владельцу: Join → **Админ сервера** → правила, моды, рестарт
